⚠️ STRATÉGIE DE TEST SELON LE TYPE DE PROBLÈME:

TYPE: SYNTAX
─ Tests critiques: Que le code S'EXÉCUTE sans crash
─ Vérifie: Pas d'ImportError, SyntaxError, NameError
─ Exemple: test_module_imports(), test_functions_exist()
─ Tolère: Logique incomplète (syntax > logique)

TYPE: LOGIC
─ Tests critiques: Le COMPORTEMENT ATTENDU est correct
─ Vérifie: Chaque fonction fait ce que son NOM dit
─ Exemple: test_calculate_discount_logic(), test_edge_cases()
─ Incluez: Tests avec valeurs limites, None, listes vides
─ N'acceptez: Que si majorité des tests passent (≥85%)

TYPE: NAMING
─ Tests critiques: Noms correspondent au comportement
─ Vérifie: Les fonctions font ce que leurs noms indiquent
─ Exemple: test_function_names_match_behavior()
─ Note: Les noms SONT la spécification

TYPE: DOCUMENTATION
─ Tests critiques: Docstrings complètes et valides
─ Vérifie: Format Google Style, Args/Returns présents
─ Exemple: test_docstrings_exist(), test_docstring_format()

CRITÈRES VALIDATION (TOUS requis):
─────────────────────────────────────────────────────────────────
1. Tests: Majorité passent (≥85% si LOGIC type, ≥90% sinon)
2. Qualité: Score Pylint ≥7.0 (ou ≥6.5 après itération 8)
3. Bugs critiques: Aucun (ImportError, SyntaxError, NameError)
